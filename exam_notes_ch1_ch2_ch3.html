Introduction to C#

Syntax · Variables · Constants · Input · Comments

What is C#?
Pronunciation"C-Sharp" — by Microsoft, runs on .NET Framework.
FamilyFrom C family. Similar to C++ and Java. Object-Oriented (OOP).
Used ForMobile, desktop, web apps, games, databases.
Basic Program Structure
using System;
namespace HelloWorld {
  class Program {
    static void Main(string[] args) {
      Console.WriteLine("Hello World!");
    }
  }
}
  • using System — access built-in classes (Console)
  • namespace — organises your code
  • class — all code must be inside a class
  • Main() — entry point, runs first
  • Statements end with a semicolon ;
  • C# is case-sensitive: MyVar ≠ myvar

Constants & Identifier Rules
Variable vs constVariable: value can change. const: value is fixed, read-only, must assign at declaration.
const int MAX = 100;   // OK
MAX = 200;             // ERROR
⚠ const must be assigned at declaration — cannot declare it empty.
  • Letters, digits, underscore _ only — no spaces
  • Must begin with a letter (not digit)
  • Case-sensitive: myVar ≠ MyVar
  • No reserved keywords: int, double, string…
  • Use descriptive names: minutesPerHour

Selection

if · else · else if · switch

Overview
ifRun block if condition is TRUE.
elseRun block if condition is FALSE.
else ifTest new condition when first is false.
switchMatch value against multiple cases.
if Statement & if/else Statement
// if syntax
if (condition)
{
    // runs if TRUE
}

// if/else syntax
if (condition)
{ /* TRUE */ }
else
{ /* FALSE */ }
// if example
int x=20, y=18;
if (x > y)
{
    Console.WriteLine("x > y");
}
// Output: x > y

// if/else example
int time=20;
if (time < 18) { Console.WriteLine("Good day."); }
else           { Console.WriteLine("Good evening."); }
// Output: Good evening.
else if Statement
// Syntax
if (cond1)      { /* cond1 true */ }
else if (cond2) { /* cond2 true */ }
else            { /* both false */ }
int time=22;
if (time < 10)       { WriteLine("Good morning."); }
else if (time < 20)  { WriteLine("Good day.");     }
else                 { WriteLine("Good evening.");  }
// Output: Good evening.
ℹ Conditions tested top-to-bottom. Once one is true, the rest are skipped.
switch Statement
switch(expression)
{
    case x:
        // code
        break;
    case y:
        // code
        break;
    default:
        // fallback
        break;
}
int day=4;
switch(day)
{
    case 1: Console.WriteLine("Mon"); break;
    case 2: Console.WriteLine("Tue"); break;
    case 3: Console.WriteLine("Wed"); break;
    case 4: Console.WriteLine("Thu"); break;
    default: Console.WriteLine("?"); break;
}
// Output: Thu
KeywordPurpose
breakExits the switch block after a match. Without it, code falls through to the next case.
defaultOptional. Runs if no case matches — like a final else.
if vs switch & Key Reminders
Use if/else when…Use switch when…
Checking a range (score > 90)Exact value match
Two outcomes (true/false)Many discrete values
Complex boolean conditionsCleaner than many else if
  • == compares · = assigns — never mix
  • else if only runs if prior conditions are false
  • Every case in switch needs a break;
  • default is optional but good practice
  • Conditions must be true or false
  • Always use {} to define each block body
